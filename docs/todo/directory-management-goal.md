# /goal: 为 AIMD Desktop 增加目录管理功能

## 背景

AIMD Desktop 当前以单文档打开、编辑、保存、导出为核心，已有最近打开、拖拽打开、多窗口、资源面板和大纲面板。下一步需要加入轻量的本地目录管理能力，让用户可以把一组 `.aimd` / Markdown 文档作为一个目录空间浏览和组织。

这个功能不是知识库、同步盘、协作系统，也不是数据库驱动的文档管理平台。第一版目标是补齐桌面应用应有的本地文件组织体验：打开目录、浏览支持的文档、创建/重命名/移动/删除文档和文件夹，并和现有单文档编辑流程打通。

## 目标

实现一个本地目录管理 MVP：

- 用户可以选择一个本地文件夹作为当前工作目录。
- 左侧栏可以在“目录 / 大纲 / 资源”之间清晰承载不同信息，不要把所有内容堆在一起。
- 目录树只展示 AIMD 支持的文档和子目录：
  - `.aimd`
  - `.md`
  - `.markdown`
  - `.mdx`
- 点击目录树中的文档，按现有 `routeOpenedPath` / 文档生命周期逻辑打开。
- 可以在当前目录或子目录中新建文档、新建文件夹、重命名、移动到废纸篓或删除。
- 文件系统变化后，目录树能刷新并保持基本选择状态。
- 最近打开仍然保留，但目录管理应成为打开项目文件夹后的主入口。

## 非目标

- 不做云同步、账号、团队协作。
- 不做全文搜索索引或知识库关系图。
- 不做 Git 状态、版本历史、冲突合并。
- 不修改 `.aimd` 容器格式来保存目录信息。
- 不把普通图片、PDF、Office 文件纳入第一版目录树；如用户拖入或通过文档引用资源，仍走现有资源逻辑。

## 设计约束

- 目录管理是本地文件系统视图，不是 AIMD 包内的虚拟目录。
- 文件操作必须经过 Tauri 后端命令，前端不能假设浏览器环境可直接访问任意路径。
- 删除优先使用系统废纸篓；如果当前平台或 API 不支持，再提供明确的永久删除路径，并在 UI 中区分。
- 重命名和移动必须保持扩展名合法；对不支持的扩展名要阻止或提示。
- 打开新文档前必须尊重当前文档 dirty 状态，不得静默丢改动。
- 目录树不能阻塞现有大纲、资源、编辑器、导出和健康检查流程。

## 建议实施路径

### 1. 后端目录 API

在 `apps/desktop/src-tauri/src` 下新增或拆分目录模块，暴露 Tauri commands：

- `open_workspace_dir()`：弹出目录选择框，返回目录路径和初始目录树。
- `read_workspace_tree(root: String)`：读取指定根目录下的子目录和支持文档。
- `create_workspace_file(root: String, parent: String, name: String, kind: "aimd" | "markdown")`：新建文档。
- `create_workspace_folder(root: String, parent: String, name: String)`：新建文件夹。
- `rename_workspace_entry(root: String, path: String, new_name: String)`：重命名文件或文件夹。
- `trash_workspace_entry(root: String, path: String)`：移入废纸篓；失败时返回可展示错误。
- `move_workspace_entry(root: String, from: String, to_parent: String)`：移动到同一工作目录内的另一个文件夹。

后端要求：

- 所有 `path` 都必须校验在 `root` 内，禁止 `..` 或符号链接逃逸工作目录。
- 遍历目录时跳过隐藏目录、`node_modules`、`.git`、`target`、`dist` 等明显非文档目录。
- 读取失败的目录不要让整个树失败，应返回局部错误或跳过并记录可展示状态。
- 树节点 DTO 至少包含：
  - `id`
  - `name`
  - `path`
  - `kind: "folder" | "document"`
  - `format?: "aimd" | "markdown"`
  - `children?: WorkspaceTreeNode[]`
  - `modifiedAt?`

### 2. 前端状态与类型

在 `apps/desktop/src/core/types.ts` 和状态模块中补充：

- `WorkspaceTreeNode`
- `WorkspaceRoot`
- 当前选中的目录根路径
- 当前选中的树节点路径
- 展开的目录节点集合
- 目录面板加载、空状态、错误状态

持久化建议：

- 使用 `localStorage` 记住最近一个工作目录路径和展开状态。
- 应用启动时可以显示“继续打开上次目录”，但不要自动读取不存在或无权限的路径时打断启动。

### 3. 左侧栏 UI

改造 `apps/desktop/src/ui/template.ts` 和相关样式：

- 在侧边栏加入目录区，打开目录后优先展示目录树。
- 大纲和资源仍然用于当前文档；可以放在目录下方，或用顶部小型分段控件切换。
- 提供目录级操作按钮：
  - 打开目录
  - 刷新
  - 新建文档
  - 新建文件夹
- 节点右键菜单提供：
  - 打开
  - 在新窗口打开
  - 重命名
  - 新建子文档
  - 新建子文件夹
  - 删除/移到废纸篓
- 当前打开文档在目录树中需要有选中态。

UI 要求：

- 不做欢迎页式营销文案。
- 控件密度应适合桌面工具，目录树行高稳定，长文件名省略但 tooltip 或 title 可查看全名。
- 不要用大卡片堆叠目录树。
- 空状态只提供必要操作，不写大段说明。

### 4. 文件操作与现有文档生命周期集成

目录树打开文档时复用现有路径：

- `.aimd` 走 `open_aimd`
- `.md` / `.markdown` / `.mdx` 走现有 Markdown 打开逻辑
- 路由入口优先复用 `apps/desktop/src/document/lifecycle.ts` 中已有打开逻辑

新建 `.aimd` 文档时：

- 创建一个最小有效 AIMD 包，而不是空文件。
- 初始 Markdown 可以是 `# 未命名文档` 或基于文件名生成标题。

新建 Markdown 文档时：

- 写入最小 Markdown 正文。
- 打开后保持现有 `.md` 保存语义；插入图片时仍按现有“保存为 AIMD”流程。

重命名当前打开文档时：

- 成功后更新当前文档路径、标题、最近打开列表和目录树选中态。
- 失败不得丢失编辑器内容。

删除当前打开文档时：

- 如果文档有未保存修改，必须先提示保存/取消。
- 删除成功后关闭当前文档或回到空状态。

### 5. 目录刷新与错误处理

需要覆盖这些情况：

- 文件被外部程序删除后，点击打开时给出明确提示，并从目录树刷新移除。
- 目录权限不足时，展示局部错误，不崩溃。
- 重名创建或重命名时，返回明确错误。
- 移动文件夹时禁止移动到自身或自身子目录。
- 当前打开文档被移动或重命名后，状态同步到新路径。

## 测试要求

至少补充这些验证：

- Rust 后端单元测试：
  - 支持扩展名过滤。
  - 路径必须限制在 workspace root 内。
  - 创建文件/文件夹、重命名、移动、删除的失败场景。
- TypeScript 类型检查：
  - `npm run typecheck`
- 现有桌面 E2E 或 Playwright 测试：
  - 打开一个临时目录。
  - 新建 `.aimd` 文档并打开。
  - 新建子文件夹，并在其中新建 Markdown 文档。
  - 重命名当前打开文档后继续保存。
  - 删除文档后目录树刷新且编辑区不崩溃。

如果现有 E2E 对系统文件选择框不好控制，可以给测试增加 dev-only 命令或测试入口，但生产 UI 不应暴露测试后门。

## 验收标准

- 从空启动界面可以打开一个本地目录。
- 打开目录后，用户能在侧边栏看到目录树，并能打开支持的文档。
- 新建、重命名、移动、删除文件/文件夹均能反映到真实文件系统。
- 当前文档 dirty 状态不会因切换目录文档、重命名或删除而丢失。
- `.aimd` 和 Markdown 的既有打开、保存、另存为、资源检查、导出功能不回归。
- 路径安全校验到位，不能通过前端传参操作 workspace root 之外的文件。
- `npm run typecheck` 通过。
- 相关 Rust 测试通过；如新增 E2E，E2E 也应通过。
