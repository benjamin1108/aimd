# /goal: 资源目录默认隐藏，并重构设置页为通用设置

## 背景

AIMD Desktop 当前左侧栏包含目录、大纲、资源等信息。目录管理功能加入后，左侧栏的主要入口应是用户打开的本地目录和当前文档大纲；资源列表属于低频调试/管理信息，不应该在文档有图片时默认占用侧边栏空间。

当前设置页标题和导航也偏向“大模型服务设置”，但应用设置已经不只包含模型配置，还包含网页导入、界面显示、资源面板等通用偏好。需要把设置页从“模型设置页”改成“应用设置页”，并新增一个默认关闭的资源目录显示开关。

## 目标

完成一个设置驱动的资源面板显示策略：

- 资源目录/资源面板默认不显示。
- 只有用户在设置中手动开启“显示资源目录/资源面板”后，左侧栏才显示资源区。
- 开关必须持久化到应用设置文件，重启后保持用户选择。
- 设置页整体文案从“大模型服务设置”改为通用“设置”或“AIMD 设置”。
- 设置导航不再只围绕大模型，应明确拆成通用分类，例如：
  - 常规
  - AI / 模型
  - 网页导入
- 资源面板开关放在“常规”或“界面”分类中，默认关闭。
- 如果用户关闭资源面板，资源检查、图片内嵌、保存、导出等功能不能受影响；只是不在侧边栏常驻展示资源列表。

## 非目标

- 不删除资源管理能力。
- 不删除“资源检查”菜单项。
- 不改变 `.aimd` 内部资源结构。
- 不改变图片粘贴、图片去重、保存时清理未使用资源、导出 Markdown assets 的逻辑。
- 不把资源列表做成完整资产管理器；这次只控制侧边栏是否展示。

## 需要检查的现有代码面

优先检查并修改这些位置：

- `apps/desktop/src/core/settings.ts`
  - 扩展 `AppSettings`，新增 UI / sidebar 偏好字段。
  - 默认值必须让资源面板关闭。
  - 兼容旧设置文件，旧文件没有该字段时按默认值补齐。
- `apps/desktop/src/settings/main.ts`
  - 把页面标题从“大模型服务设置”改为通用设置标题。
  - 重构左侧设置导航，使模型不再是唯一主语。
  - 新增资源面板显示开关。
  - 保存、取消、恢复默认模型等按钮语义需要继续合理；不要让“恢复默认模型”出现在所有设置分类里显得像全局恢复。
- `apps/desktop/src/ui/chrome.ts`
  - 当前资源区显示逻辑是按 `doc.assets.length > 0` 自动展示，需要改为同时受设置开关控制。
  - 文档没有资源时仍不能显示空资源区。
- `apps/desktop/src/core/state.ts` / `apps/desktop/src/core/types.ts`
  - 如果需要在主窗口持有设置快照，补充类型和状态。
- `apps/desktop/src/main.ts`
  - 启动时加载设置偏好，应用到侧边栏。
  - 设置保存后如果主窗口能收到更新事件，应即时刷新资源面板；如果现有架构没有跨窗口事件，至少要求重启后生效，并在 goal 完成说明里写清楚。
- `apps/desktop/src/styles/settings.css`
  - 调整设置页分类与字段布局，保持桌面工具密度，不做营销式说明。

## 建议数据结构

建议在应用设置中增加：

```ts
type UiSettings = {
  showAssetPanel: boolean;
};

type AppSettings = {
  ai: AiSettings;
  webClip: WebClipSettings;
  ui: UiSettings;
};
```

默认值：

```ts
ui: {
  showAssetPanel: false,
}
```

命名可以按项目风格调整，但语义必须清楚：这是“是否显示侧边栏资源面板”，不是是否启用资源功能。

## UI 要求

设置页：

- 标题使用“AIMD 设置”或“设置”。
- 顶部说明不要只说“配置模型连接凭证”。
- 导航分类建议：
  - 常规：界面显示、资源面板开关。
  - AI / 模型：Provider、模型、API Key、API Base、测试连接。
  - 网页导入：是否启用智能排版、使用的模型、输出语言。
- “恢复默认模型”只在 AI / 模型分类中显示，或文案改成明确只影响模型设置。
- 资源面板开关文案建议：
  - 标题：显示资源面板
  - 说明：在左侧栏显示当前文档内嵌资源列表。关闭后不影响图片保存、资源检查和导出。

主窗口：

- 默认启动时，即使文档有图片，也不显示资源区。
- 开启设置后，有资源的文档显示资源区；无资源文档仍隐藏资源区。
- 大纲和目录的空间不应因为隐藏资源区留下空白。

## 行为要求

- 打开含图片 `.aimd` 文档：
  - 默认：侧边栏不显示资源面板。
  - 开启设置后：侧边栏显示资源面板和资源数量。
- 切换文档、保存文档、插入图片后，资源面板显示状态按设置重新计算。
- 资源检查入口仍在文档操作菜单中可用。
- 关闭资源面板不影响：
  - 图片渲染
  - 图片灯箱
  - 图片粘贴
  - 保存 `.aimd`
  - 清理未引用资源
  - 导出 Markdown + assets

## 测试要求

至少补充或更新这些测试：

- TypeScript 类型检查：
  - `npm run typecheck`
- 项目门禁：
  - `npm run check`
- 设置默认值测试或 E2E：
  - 旧设置文件没有 `ui` 字段时，默认 `showAssetPanel` 为 `false`。
  - 设置页能保存 `showAssetPanel`。
- 主窗口 E2E：
  - 打开含资源文档时，默认不显示 `#asset-section`。
  - 将设置 mock 为 `showAssetPanel: true` 后，含资源文档显示 `#asset-section`。
  - 资源检查按钮仍可点击，不依赖资源面板显示。

如现有 E2E 使用 Tauri invoke mock，需要在 mock 的 `load_settings` / `save_settings` 中覆盖新字段。

## 验收标准

- 默认设置下，资源面板不再因为文档含图片而自动出现在侧边栏。
- 用户可以在设置页手动开启资源面板显示。
- 设置页不再表现为“只有大模型设置”的页面。
- 模型配置、网页导入配置仍可正常保存和加载。
- 资源检查与资源相关文档能力不回归。
- `npm run check` 通过。
- 相关 E2E 或单元测试通过。
