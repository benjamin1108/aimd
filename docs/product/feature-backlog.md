# AIMD 大功能待办记录

更新时间：2026-05-17

这个文档只记录后续要做的大功能方向，不是 `/goal`，也不是实现方案。真正开工时再拆成具体需求、交互、技术方案和验收标准。

## 1. Git 功能补齐

围绕 Git 使用链路补齐完整能力。

- Git diff 支持左右并排对比，而不只是单栏补丁文本。
- Git 操作支持撤销，例如撤销文件改动、撤销暂存、恢复到指定版本等。
- 补齐常用 Git 操作入口，让用户不需要离开 AIMD 就能完成基础版本管理。
- Git 操作需要有清晰反馈，不能出现点了按钮没反应、失败原因不可见的情况。

## 2. 支持所有类型文本文件读取和编辑

AIMD 不只打开 `.aimd` 和 Markdown，也要能作为通用文本文件查看和编辑入口。

- 支持常见纯文本、代码、配置、日志等文本文件读取。
- 支持直接编辑和保存这些文本文件。
- 根据文件类型提供合理的展示方式，例如代码高亮、纯文本模式、配置文件模式。
- 对不确定编码、超大文本、二进制误判等情况给出清楚提示。

## 3. 增加展示阅读模式

增加一种面向展示和沉浸阅读的模式，把当前 Markdown 以更好的视觉形式展示出来，类似 PPT / 演示文稿，而不是普通编辑器里的长文档。

- 可以把当前 Markdown 自动分段、分页或分屏展示。
- 适合会议汇报、方案讲解、日报周报展示、AI 报告演示等场景。
- 展示模式应保留标题、图片、表格、代码块等内容的清晰层级。
- 用户不需要手工制作 PPT，也能把 `.aimd` / Markdown 直接拿来讲。
- 后续可以考虑主题、版式、演讲视图、全屏播放和导出展示稿。

## 4. 资源可下载

资源面板里的资源不能只是能看，还要能方便导出和下载。

- 单个资源支持下载到本地。
- 多个资源支持批量导出。
- 图片、附件、嵌入文件等资源需要保留清晰文件名和类型。
- 对缺失、损坏、外链资源要有明确状态提示。

## 5. 增加更多类型的图形资源

扩展 AIMD 对图形类资源的表达和管理能力，特别是代码相关图形。

- 支持流程图、架构图、时序图、状态图、关系图等图形资源。
- 重点支持代码类图形，例如调用关系图、模块依赖图、类图、数据流图、AST/结构图等。
- 图形资源应能被文档引用、预览、导出和管理。
- 优先考虑可编辑、可重新生成的图形描述，而不是只能保存一张死图。
