你需要先判断文档类型，然后选择对应的导读策略。

文档类型判断规则：

1. 如果文档是报告、分析、研究、调研、复盘：
   导读重点是：
   - 核心结论是什么；
   - 论据如何支撑结论；
   - 哪些数据、图表、表格值得注意；
   - 有哪些风险、限制或不确定性；
   - 最后读者应该带走什么判断。

2. 如果文档是教程、说明书、操作指南：
   导读重点是：
   - 使用前需要知道什么；
   - 操作流程的关键节点；
   - 哪些步骤容易出错；
   - 每一步完成后应该得到什么结果；
   - 读者如何判断自己做对了。

3. 如果文档是产品需求、PRD、功能设计、技术方案：
   导读重点是：
   - 要解决的用户问题是什么；
   - 核心流程如何运转；
   - 关键设计决策是什么；
   - 有哪些约束、边界和非目标；
   - 哪些部分影响实现成本或用户体验。

4. 如果文档是会议纪要、项目同步、工作记录：
   导读重点是：
   - 已经做出的决定；
   - 谁负责什么；
   - 时间点和交付物；
   - 未解决的问题；
   - 下一步行动。

5. 如果文档是论文、学术笔记、技术研究：
   导读重点是：
   - 研究问题是什么；
   - 方法或实验设计是什么；
   - 主要发现是什么；
   - 证据是否充分；
   - 有哪些局限和后续问题。

6. 如果文档是商业计划、融资材料、市场材料：
   导读重点是：
   - 面向谁；
   - 痛点是什么；
   - 方案为什么成立；
   - 商业价值或市场机会在哪里；
   - 风险和可信度如何。

7. 如果文档类型不明确：
   使用通用信息架构导读：
   - 先抓主旨；
   - 再抓结构；
   - 再抓关键证据；
   - 再抓风险或限制；
   - 最后抓结论和下一步。

导读质量要求：

- 每一步都要有信息密度。
- 不要复述标题。
- 不要把每一步都写成一样的句式。
- 不要只说“这一节介绍了什么”，要说明“读者应该从这里看出什么”。
- 如果有数字、表格、结论、风险、决策，要优先讲。
- 如果是流程类文档，要讲清楚前后依赖。
- 如果是分析类文档，要讲清楚观点和证据。
- 如果是设计类文档，要讲清楚取舍和边界。
- 如果是会议类文档，要讲清楚决定、责任人和下一步。
